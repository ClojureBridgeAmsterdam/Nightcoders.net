The title element in the <head></head>-part of the HTML definition is responsibile for creating the title of the web page (this is agreed upon in the HTML standard that web browsers implement). Look in index.html!
  4. Change the text on the title from "Hello world" to "hello <your name>"

    Here we change the text string we pass to greeting. greeting is a function. By putting the brackets around it we "call" that function (same as we showed earlier in the REPL with (+ 1 1 1) ;; => 3, where + is called with the arguments 1, 1 and 1). The function will evaluate itself (check what code it has in its body) with the arguments (the text string) passed. Substituting the parameter in the body of the function with that what was passed.

    In our case:

    (defn greeting [message]
  [:h1 message])

    (Here defn means we define a new function.) Calling that function as for example [greeting "Hello"] turns into: 

    [:h1 "Hello"]

    And this means that on the web page we see <h1>Hello</h1>. And finally that is interpreted as a large header (h1) with the text "Hello".

    Phew!

  8. Let's add some flags to the clocks! Look at line 63 of the code. Uncomment (in Clojure code that is after ; is ignored by the compiler) and enjoy.
  9. Look at city-data, what does it mean? What do the curly braces mean? 
    (def city-data
  {"Amsterdam"     {:time-diff  0 :flag-url (str base-url "netherlands.png")}
    "Paris"        {:time-diff  0 :flag-url (str base-url "france.png")}
    ...}

    Clojure has four main types of data structures: the list like (+ 1 1 1) (between parentheses), which is interpreted as a function call; the vector like [1 2 3 3 3] (between brackets), that is a list of items; a set like #{"yoga" "meditation"} (in #{}) that contains various items without duplicates, and the map like above (between curly braces) that contain key value associations.

    Beside these there are atomic data types like integers as 12, doubles like 1.22, strings like "Hello", characters like \a \b \c, symbols like greeting, null being nil, booleans like true and keywords like :time-diff. They all evaluate to themselves, except symbols that point to something else (like greeting points to the function with the name greeting).

Fourth part: advanced

  • Get a list of all flag-urls from city-data First get the values of the map (the data structure with key value pairs) using the function vals. Then use the function map on the result to get the :time-diff out of each element.
    Know that a keyword like :time-diff can be used as a function that looks itself up in the map data structure.
  • Sort the list of cities and times Use sort.
    • first alphabetically
    • Then by time For this the function sort-by can be used. sort-by is a function that sorts a sequence (second argument) using a function (first argument).
      And know that keywords like :time-diff can be used as a function.
  • Find the average time-difference Get all time-differences, for example with (map :time-diff (vals city-data)).
    Then get the average of it by writing your own average function (there is not one build in in Clojure). Click here for help for writing the function for average (also called mean).
